# weapon-range-damage-viewer 规范

## 目的

在保留地图测距的前提下，依据 `public/weapon` 下由构建脚本从 xlsx **Raw Values** 表导出的 **JSON**（默认 `weapons.json`，由 `manifest.json` 的 `weaponsJson` 指定）中的武器数值（含每条记录的 `columns` 全列及 `displayName` / `weaponType` 等），按约定公式计算并展示各部位在不同距离下的伤害及护甲相关数值。更新表格源文件后须重新运行 `npm run weapon:json`（或 `npm run weapon:data`）再部署或刷新静态资源。

## 需求

### 需求:入口与页面布局

系统必须在 **Vue 应用内**提供导航入口（例如首页或站点顶栏中的链接），使用户可进入武器距离伤害工具视图。武器工具必须为 **Vue 路由视图**（不得再依赖 `pages/` 下独立 HTML 文件作为唯一入口）。界面必须使用 **Reka UI** 实现表单与交互控件（替代 Pico CSS CDN）；允许项目级样式在 Reka UI 之上统一视觉。页面布局必须与地图工具视图同型：左侧为正方形地图视口（与地图工具相同的缩放、平移与测距能力），右侧为信息栏。右侧信息栏在水平方向上的占宽规则必须与地图工具视图一致，并满足 `map-range-viewer` 中「页面布局与地图视口尺寸」需求对**右栏占满除左侧地图以外剩余宽度**的约束。

#### 场景:从首页进入

- **当** 用户在应用内首页点击「武器距离伤害」或等价链接
- **那么** 必须进入武器工具路由且左侧地图与右侧栏可见

#### 场景:宽屏下右栏与地图页一致占满剩余宽度

- **当** 用户在宽屏桌面浏览器中打开武器工具视图且视口宽度足以并排显示左侧地图与右侧栏
- **那么** 右侧信息栏必须在水平方向上占满除左侧正方形地图以外的**全部**剩余可用空间，不得在信息栏右侧因栏宽上限留下大块空白

### 需求:地图选择与距离

系统必须在右侧栏最上方提供地图下拉选择，且必须与现有七张竞技地图数据源一致。系统必须在与地图工具相同的语义下展示**当前测距距离**（游戏单位），该距离必须驱动下方武器伤害计算中的「距离」变量。在地图覆盖层上，在已确定测距起点后，系统**不得**绘制以该距离为半径、以起点为圆心的**整圆**来表示测距长度；系统**必须**绘制**从测距起点到当前测距终点**的**线段**（未固定终点时终点随光标，固定后终点为第二次点击位置），线段样式须清晰可见且与页面现有测距主题一致；测距起点**必须**保留可见锚点标记（例如小圆点，以实现为准）。

#### 场景:距离参与计算

- **当** 用户在地图上完成测距且距离有效
- **那么** 武器伤害表必须使用当前距离值代入公式中的指数项

#### 场景:线段表示测距且无半径圆

- **当** 用户已放置测距起点且当前存在有效终点（光标或已固定）
- **那么** 覆盖层必须显示从起点到终点的线段，且不得显示表示该距离的半径圆

### 需求:武器 Tag 分组多选与批量操作

系统必须将武器选择控件实现为按 `weaponType` 分组的 Tag 区域。每个武器类型必须独立成一行（例如第一行为 `Pistol`）。每行第一列必须提供批量按钮：当该类型尚未全选时，按钮文案必须显示 `All` + 该类型复数形式；当该类型已全选时，按钮文案必须显示 `Clear` + 该类型复数形式。点击批量按钮时，系统必须对该类型执行全选或全清。

#### 场景:按类型分组显示 Tag

- **当** 页面成功加载武器数据
- **那么** 武器选择区必须按 `weaponType` 分行展示，且每把武器为可点击 Tag

#### 场景:批量按钮文案切换

- **当** 某类型下并非全部武器被选中
- **那么** 该类型行首按钮必须显示 `All` + 类型复数形式
- **并且** 当该类型下全部武器被选中时，该按钮必须切换为 `Clear` + 类型复数形式

#### 场景:批量全选与全清

- **当** 用户点击某类型行首 `All...` 按钮
- **那么** 该类型下全部武器 Tag 必须变为选中状态
- **并且** 当用户点击 `Clear...` 按钮时，该类型下全部武器 Tag 必须清除选中状态

### 需求:武器 JSON 数据源（由 xlsx 预生成）

武器原始表为 xlsx 内 **`Raw Values`** 工作表（名称匹配策略以实现为准）。**运行时**页面必须仅通过 HTTP 加载 `public/weapon/` 下由 **`manifest.json`** 的 `weaponsJson` 字段指定的 JSON 文件（缺省时为 `weapons.json`）。该 JSON 必须由仓库内脚本（`scripts/weapon-xlsx-to-json.mjs`）从 `manifest.json` 所指定 xlsx 生成。

每条武器记录**必须**满足：

- **`id`**：字符串，取自 **A 列单元格显示文本**（trim），作为稳定标识与选项值，与现有计算查找一致。
- **`displayName`**：字符串，用于 Tag 标签等 UI 展示；**必须**优先采用 **A 列对应单元格的注释（comment）文本**；若该单元格无注释或注释为空，**必须**回退为 **A 列单元格显示文本**（与 `id` 一致或等价）。
- **`weaponType`**：字符串，表示武器类型（例如 `pistol`），语义须与源表分类一致；具体从 xlsx 的列或表内分段规则导出，以实现与源表对齐为准。
- **全列数据**：对 Raw Values 表**首行表头**所定义的**每一列**（除实现中明确跳过的全空列外），必须在记录中持久化该列对该行的值；结构化形式**必须**为键值对象（或等价结构），键为表头（trim），值为自单元格导出的标量（数字/字符串/布尔/null 等与导出逻辑一致）。公式计算所需的 `Damage`、`HeadshotMultiplier`、`RangeModifier`、`WeaporArmorRatio`（及脚本已支持的列名别名）**必须**能从该结构中解析得到；若某列在表头中缺失或该行无效，则该行不得作为有效武器条目输出。

#### 场景:manifest 与 JSON 就绪

- **当** `manifest.json` 存在且 `weaponsJson` 指向的 JSON 已生成并包含至少一把有效武器
- **那么** 页面必须成功加载该 JSON 且能列出武器

#### 场景:武器展示名使用注释

- **当** 某武器 A 列单元格存在非空注释
- **那么** 武器 Tag（或等价列表）中该条目的可见标签必须显示注释文本，且选项值仍必须对应稳定 `id`

#### 场景:无注释时使用单元格文本

- **当** 某武器 A 列单元格无注释或注释为空
- **那么** `displayName` 与可见标签必须使用 A 列单元格文本

#### 场景:加载或格式失败

- **当** JSON 缺失、无法请求或内容不符合约定结构
- **那么** 系统必须在右栏展示可读错误信息（可提示运行 `npm run weapon:json`），且不得抛出未捕获异常导致白屏

### 需求:伤害指数因子

设 \(d\) 为当前距离（游戏单位），`RangeModifier` 为表中取值。系统必须计算 \(R = \text{RangeModifier}^{(d/500)}\)（即 `RangeModifier` 的 \((d/500)\) 次幂），且所有下列部位公式中的「距离衰减部分」必须使用同一 \(R\)。

#### 场景:距离为 0 或正值

- **当** 距离 \(d\) 为非负有限数且 `RangeModifier` 为正有限数
- **那么** \(R\) 必须为有限正数并参与各部位计算

### 需求:无护甲伤害（表格第 3 行第 2–5 列）

系统必须按下式计算无护甲伤害（结果展示保留 **2 位小数**），并将结果乘以 `bullets`：

- **头**：\((\text{Damage} \times \text{HeadShotMultiplier} \times R) \times \text{bullets}\)
- **胸/手臂**：\((\text{Damage} \times 1.00 \times R) \times \text{bullets}\)
- **腹**：\((\text{Damage} \times 1.25 \times R) \times \text{bullets}\)
- **腿**：\((\text{Damage} \times 0.75 \times R) \times \text{bullets}\)

其中，若当前测距距离 `d` 严格大于该武器的 `range`，系统必须将以上四列无护甲伤害全部显示为 `0.00`。

#### 场景:切换武器或距离

- **当** 用户更换武器或地图测距距离变化
- **那么** 第 3 行第 2–5 列必须按上式（含 `bullets`）更新

#### 场景:超出射程时无护甲归零

- **当** 当前距离 `d` 大于所选武器 `range`
- **那么** 第 3 行第 2–5 列必须全部显示 `0.00`

### 需求:有护甲伤害（表格第 3 行第 6–8 列）

系统必须仅在三列展示：**头、胸/手臂、腹**（与表格列对齐）。有护甲伤害必须在对应无护甲基础公式中同时乘以 \((\text{WeaponArmor}/2)\) 与 `bullets`。展示格式必须为 `XX.XX(XX)`：前半部分为有护甲伤害（保留 **2 位小数**），括号内为对应部位的护甲减少整数值（按护甲减少公式计算并取整）。

- **头（有护甲）**：\(((\text{Damage} \times \text{HeadShotMultiplier} \times R) \times (\text{WeaponArmor}/2)) \times \text{bullets}\)
- **胸/手臂（有护甲）**：\(((\text{Damage} \times 1.00 \times R) \times (\text{WeaponArmor}/2)) \times \text{bullets}\)
- **腹（有护甲）**：\(((\text{Damage} \times 1.25 \times R) \times (\text{WeaponArmor}/2)) \times \text{bullets}\)

其中，若当前测距距离 `d` 严格大于该武器的 `range`，系统必须将三列有护甲伤害显示为 `0.00(0)`。

#### 场景:有护甲三列更新

- **当** 武器或距离变化
- **那么** 有护甲三列必须按上式（含 `bullets`）更新，且每个单元格必须展示为 `XX.XX(XX)`

#### 场景:超出射程时有护甲归零

- **当** 当前距离 `d` 大于所选武器 `range`
- **那么** 有护甲三列必须全部显示为 `0.00(0)`

### 需求:高伤害阈值红色高亮

系统必须对结果表格中的伤害数值执行阈值高亮：当任一伤害单元格的展示值大于或等于 100 时，该数值文本必须以红色字体显示；小于 100 时必须保持默认文本颜色。

#### 场景:达到阈值显示红色

- **当** 任一武器任一伤害列的展示值计算结果为 100 或更高
- **那么** 对应单元格中的数值文本必须显示为红色

#### 场景:低于阈值不高亮

- **当** 任一武器任一伤害列的展示值计算结果低于 100
- **那么** 对应单元格中的数值文本不得使用红色高亮样式

### 需求:表格结构

系统必须将结果表格实现为“表头 + 动态数据行”结构。表头第一列必须为 **Weapon**，其后列按“无护甲（头、胸/手臂、腹、腿）+ 有护甲（头、胸/手臂、腹）”排列。每个被选中的武器必须对应一条数据行，且该行第一列显示武器展示名称。系统必须在武器选中状态变化时同步增删对应数据行。

#### 场景:选中武器新增行

- **当** 用户选中一个未选中的武器 Tag
- **那么** 结果表必须新增且仅新增该武器对应的一行

#### 场景:取消选中武器删除行

- **当** 用户取消选中一个已选中的武器 Tag
- **那么** 结果表必须删除该武器对应的一行，其他武器行保持不变

### 需求:武器选择区可折叠展示

系统必须将按 `weaponType` 分组的武器 Tag 多选区域（含每行批量按钮与 Tag 列表）封装为独立 UI 单元（组件或等价结构）。该单元必须支持展开与折叠两种状态。折叠状态下，系统必须仅展示固定文案「选择武器」以及表示可向下展开的图标（或等价视觉），不得占用展开态下 Tag 列表的纵向空间。展开状态下，系统必须继续满足「武器 Tag 分组多选与批量操作」需求的全部场景。用户必须能够通过点击标题行（或专用控件）在折叠与展开之间切换。

#### 场景:折叠时仅显示标题与展开 affordance

- **当** 用户将武器选择区置于折叠状态
- **那么** 界面必须显示「选择武器」文案与向下展开图标（或等价）
- **并且** 不得显示各 `weaponType` 行内的武器 Tag 与批量按钮

#### 场景:展开时行为与现有一致

- **当** 用户将武器选择区置于展开状态
- **那么** 武器必须仍按 `weaponType` 分行展示，且每行批量按钮与 Tag 多选行为必须符合「武器 Tag 分组多选与批量操作」中的场景

### 需求:表格武器名可打开详情模态

系统必须使伤害结果表格中每一数据行第一列（武器展示名称）成为可激活控件。当用户激活该控件时，系统必须打开模态窗口（对话框）。模态必须从已加载的武器 JSON 记录中读取该武器数据并展示静态属性；不得依赖额外网络请求。模态必须提供关闭方式（例如关闭按钮、点击遮罩或 Escape，以实现与 Reka UI 一致为准）。除第一列外，表格其余伤害数值单元格不得因本需求变为打开同一模态的入口。

#### 场景:点击武器名打开模态

- **当** 用户在结果表中点击某一行的武器名称控件
- **那么** 系统必须显示模态窗口
- **并且** 模态内容必须对应该行所选武器

#### 场景:关闭模态

- **当** 用户以组件支持的关闭方式关闭模态
- **那么** 模态必须消失且测距与武器选中状态保持不变

### 需求:武器详情模态中的 JSON 字段展示

模态内必须按下列规则展示属性（数值来自该武器记录的 `columns` 或等价结构，键名匹配须大小写不敏感；缺失或非有限数时展示「—」或省略该项，但不得崩溃）：

- **护甲衰减**：`WeaporArmorRatio`（或实现中已支持的同义列）除以 2，结果以百分比形式展示。
- **基本伤害**：`Damage`。
- **距离衰减**：`RangeModifier`。
- **爆头倍率**：`HeadshotMultiplier`（或实现中已支持的同义列名）。
- **射速**：\(60 / \text{CycleTime}\)，单位为 rpm（`CycleTime` 须为正有限数）。
- **击杀奖励**：以货币形式展示，前缀为「$」加上 `KillAward` 的数值。
- **最大速度**：`MaxPlayerSpeed`，单位为 unit/s。
- **静步最大速度**：`MaxPlayerSpeed × 0.52`，单位为 unit/s。
- **蹲走最大速度**：`MaxPlayerSpeed × 0.34`，单位为 unit/s。
- **站立 / 蹲 / 移动（精度与精确射击距离）**：分别对应 `InaccuracyStand`、`InaccuracyCrouch`、`InaccuracyMove`。对每一姿态，系统必须先将该姿态的 `Inaccuracy` 与同条记录 `columns` 中的 **`Spread`** 相加得到有效不准确度 \(\text{EffectiveInaccuracy}\)（若 `Spread` 缺失、为空或非有限数，则 **`Spread` 按 0** 参与求和）。精度为 \(\text{EffectiveInaccuracy}/1000\) rad；精确射击距离为 \(152.4/\text{EffectiveInaccuracy}\) m（`EffectiveInaccuracy` 须为正有限数；否则该项展示「—」）。
- **弹夹容量**：来自表内弹容列（例如 `clip_size` 或导出脚本使用的等价列名）。
- **备弹**：展示 `m_nPrimaryReserveAmmoMax` 的数值；若 `m_bReserveAmmoAsClips` 为 true，则该数值的单位文案为「弹夹」，否则为「子弹」。
- **武器价格**：`WeaponPrice`。
- **最远射击距离**：`Range`。
- **类型**：`WeaponType`（列内字符串）。
- **霰弹数**：仅当 `Bullets` 严格大于 1 时展示该项及其标签；为 1 或缺失时不展示霰弹数行。

#### 场景:模态展示与 JSON 一致

- **当** 用户打开某武器的详情模态且该武器在 JSON 中具有上述列的有效取值
- **那么** 模态必须包含上述各字段的格式化展示，且射速、速度、百分比与精度公式等必须符合本节说明

#### 场景:单弹丸武器不展示霰弹数

- **当** 某武器的 `Bullets` 为 1 或缺失
- **那么** 模态不得展示「霰弹数」条目

#### 场景:多弹丸武器展示霰弹数

- **当** 某武器的 `Bullets` 为大于 1 的有限数
- **那么** 模态必须展示霰弹数及其数值
