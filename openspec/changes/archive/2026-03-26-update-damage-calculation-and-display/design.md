## Context

现有 `weapon-range-damage-viewer` 已具备按距离计算伤害并展示表格的能力，但当前规则未覆盖三项业务约束：高伤害显著标识、总伤害与 `bullets` 关联、超出武器射程后的归零逻辑。此次变更不引入新页面和新数据源，仅在现有计算与渲染链路上做规则增强。

## Goals / Non-Goals

**Goals:**
- 统一伤害计算口径：基础部位伤害先按距离衰减计算，再按 `bullets` 进行总伤害汇总。
- 增加射程保护：当当前测距距离大于武器 `range` 时，所有展示伤害均为 0。
- 增强可视化可读性：表格中伤害值大于或等于 100 的数值使用红色字体。

**Non-Goals:**
- 不修改地图测距交互与距离采集方式。
- 不调整武器数据导入流程（xlsx -> JSON）。
- 不引入新的外部库或后端接口。

## Decisions

1. 计算顺序固定为「距离判定 -> 距离衰减计算 -> bullets 乘算 -> 展示格式化」。
   - 理由：超距归零是最高优先级业务规则，提前判定可避免无意义计算和格式化。
   - 备选方案：先算完再覆盖为 0。该方式实现更分散，易遗漏某些列。

2. `bullets` 作为总伤害乘数应用于无护甲与有护甲列。
   - 理由：需求描述为“计算伤害时乘以 bullets”，应作用于最终展示伤害结果。
   - 备选方案：仅作用于无护甲列。该方式会造成两组结果口径不一致。

3. 高伤害高亮基于展示值阈值（>=100）进行条件样式渲染。
   - 理由：与用户感知直接一致，不依赖内部中间值。
   - 备选方案：基于未格式化中间值判断。会出现显示值与高亮状态不一致边界问题。

## Risks / Trade-offs

- [风险] 旧数据中 `bullets` 或 `range` 缺失导致 NaN 或错误分支 -> [缓解] 缺失时按安全默认值处理并在开发期日志告警。
- [权衡] 红色高亮强化了视觉提示，但可能与主题色冲突 -> [缓解] 使用现有主题可访问颜色变量，保证对比度。
- [风险] 变更涉及所有伤害列，回归范围较大 -> [缓解] 增加针对边界值（99.99/100、range±1）的测试用例。
