## 上下文

现有 `map-range-viewer` 已具备距离、基础伤害、红/绿参考圈与地图交互，但右侧信息区没有玩家状态参数（HP/护甲）与爆炸后结果，用户只能看到“伤害值”，无法直接得到“是否存活、剩余护甲”。本变更在不改变地图交互主流程的前提下，为伤害结果增加可调输入与结算输出。

## 目标 / 非目标

**目标：**

- 在“计算伤害”下方新增两个滑块：初始 HP、初始护甲，范围 0-100、步长 1。
- 新增“爆炸后状态”输出行：`HP:xx.xx (ceil)，护甲：xx`。
- 按指定规则结算 HP/护甲扣减，并驱动显示颜色（HP<0 时红色固定为 `HP:0(0)，护甲：0`）。
- 将红圈阈值从固定 damage=100 改为由滑块状态驱动：
  - HP >= 护甲时目标伤害 `T = HP + 护甲`
  - HP < 护甲时目标伤害 `T = HP * 2`
- 红圈反解半径必须与绿圈「有效伤害边界」一致：因 `d > 3.5*baseDamage` 时伤害为 0，红圈半径不得大于 `3.5*baseDamage`（与绿圈同半径上限），反解值过大时钳制。

**非目标：**

- 不修改地图缩放/平移/测距核心交互。
- 不引入新框架与后端接口。

## 决策

| 决策 | 说明 | 备选 |
| --- | --- | --- |
| 滑块状态源 | 使用前端内存状态（两个 number）并实时触发重算 | URL 持久化非本次范围 |
| 结果显示格式 | HP 保留两位小数并附 `ceil`，护甲显示整数 | 统一整数会损失信息 |
| 护甲扣减规则 | 完全按需求分支实现（`armor > damage*0.5`） | 不做“游戏近似优化” |
| 红圈反解方式 | 基于伤害公式反推距离：`damage = base * exp(-pow((6d)/(7base),2)/2)`，按目标伤害 `T` 求 `d` | 二分迭代求解也可，但闭式解更稳定 |

## 风险 / 权衡

- **[风险]** 目标伤害 `T` 可能超出可达区间（如 `T > baseDamage` 或 `T <= 0`）→ **缓解**：红圈不绘制并显示空状态/保持已定义的安全降级逻辑。
- **[风险]** 结算展示中“HP<0 显示 0(0)”与内部浮点值不同步 → **缓解**：将显示值与颜色逻辑统一在同一函数中生成。
- **[权衡]** 实时拖动滑块会频繁重绘 → 由于画布简单、元素少，直接重算可接受。

## 迁移计划

仅前端静态页面更新；无数据迁移。回滚为撤销 `public/map-range.html` 对滑块与结算逻辑的改动。

## 开放问题

- 无。当前规则已由需求明确给出并可直接实现。
