## MODIFIED Requirements

### 需求:地图选择与距离

系统必须在右侧栏最上方提供地图下拉选择，且必须与现有七张竞技地图数据源一致。系统必须在与地图工具相同的语义下展示**当前测距距离**（游戏单位），该距离必须驱动下方武器伤害计算中的「距离」变量。在地图覆盖层上，在已确定测距起点后，系统**不得**绘制以该距离为半径、以起点为圆心的**整圆**来表示测距长度；系统**必须**绘制**从测距起点到当前测距终点**的**线段**（未固定终点时终点随光标，固定后终点为第二次点击位置），线段样式须清晰可见且与页面现有测距主题一致；测距起点**必须**保留可见锚点标记（例如小圆点，以实现为准）。

#### 场景:距离参与计算

- **当** 用户在地图上完成测距且距离有效
- **那么** 武器伤害表必须使用当前距离值代入公式中的指数项

#### 场景:线段表示测距且无半径圆

- **当** 用户已放置测距起点且当前存在有效终点（光标或已固定）
- **那么** 覆盖层必须显示从起点到终点的线段，且不得显示表示该距离的半径圆

### 需求:武器 JSON 数据源（由 xlsx 预生成）

武器原始表为 xlsx 内 **`Raw Values`** 工作表（名称匹配策略以实现为准）。**运行时**页面必须仅通过 HTTP 加载 `public/weapon/` 下由 **`manifest.json`** 的 `weaponsJson` 字段指定的 JSON 文件（缺省时为 `weapons.json`）。该 JSON 必须由仓库内脚本（`scripts/weapon-xlsx-to-json.mjs`）从 `manifest.json` 所指定 xlsx 生成。

每条武器记录**必须**满足：

- **`id`**：字符串，取自 **A 列单元格显示文本**（trim），作为稳定标识与选项值，与现有计算查找一致。
- **`displayName`**：字符串，用于下拉等 UI 展示；**必须**优先采用 **A 列对应单元格的注释（comment）文本**；若该单元格无注释或注释为空，**必须**回退为 **A 列单元格显示文本**（与 `id` 一致或等价）。
- **`weaponType`**：字符串，表示武器类型（例如 `pistol`），语义须与源表分类一致；具体从 xlsx 的列或表内分段规则导出，以实现与源表对齐为准。
- **全列数据**：对 Raw Values 表**首行表头**所定义的**每一列**（除实现中明确跳过的全空列外），必须在记录中持久化该列对该行的值；结构化形式**必须**为键值对象（或等价结构），键为表头（trim），值为自单元格导出的标量（数字/字符串/布尔/null 等与导出逻辑一致）。公式计算所需的 `Damage`、`HeadshotMultiplier`、`RangeModifier`、`WeaporArmorRatio`（及脚本已支持的列名别名）**必须**能从该结构中解析得到；若某列在表头中缺失或该行无效，则该行不得作为有效武器条目输出。

#### 场景:manifest 与 JSON 就绪

- **当** `manifest.json` 存在且 `weaponsJson` 指向的 JSON 已生成并包含至少一把有效武器
- **那么** 页面必须成功加载该 JSON 且能列出武器

#### 场景:武器展示名使用注释

- **当** 某武器 A 列单元格存在非空注释
- **那么** 武器下拉（或等价列表）中该条目的可见标签必须显示注释文本，且选项值仍必须对应稳定 `id`

#### 场景:无注释时使用单元格文本

- **当** 某武器 A 列单元格无注释或注释为空
- **那么** `displayName` 与列表可见标签必须使用 A 列单元格文本

#### 场景:加载或格式失败

- **当** JSON 缺失、无法请求或内容不符合约定结构
- **那么** 系统必须在右栏展示可读错误信息（可提示运行 `npm run weapon:json`），且不得抛出未捕获异常导致白屏
