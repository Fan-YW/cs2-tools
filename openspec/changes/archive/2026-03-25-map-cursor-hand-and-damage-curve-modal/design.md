## 上下文

`public/map-range.html` 已具备地图放大/平移/测距圆与 C4 伤害公式展示能力，但缺少：

- 更明确的“左键长按进入平移”的鼠标光标反馈。
- 以图形方式直观呈现“伤害随距离”的衰减曲线（以及悬浮读数/首点点击标记）。
- 地图视口左侧/上侧的刻度坐标标注（需要从对应地图 `.txt` 的 `pos_x/pos_y` 获取左上世界坐标，并随缩放变化生成主/副刻度）。

本变更在不引入新框架的前提下，扩展单页的前端交互与绘制能力。

## 目标 / 非目标

**目标：**

1. 地图区域左键长按：将光标从 `crosshair` 切换为“抓手/手型”（平移预期更明显）。
2. 坐标刻度与标注：
   - 解析 `public/map/<mapName>.txt` 的 `pos_x`、`pos_y`，作为左上角世界坐标。
   - 根据当前放大倍率计算主刻度与副刻度大小。
   - 主刻度取值为 `1/2/5 * 10^n`（其中 `n` 为整数，按距离/缩放自动选择最合适的一档）。
   - 当主刻度取值以 `1` 或 `5` 开头时：每段有 `4` 个副刻度（major/5）。
   - 当主刻度取值以 `2` 开头时：每段有 `3` 个副刻度（major/4）。
   - 在地图左边与上边绘制主/副刻度与主刻度的坐标数值。
3. 右侧“显示伤害曲线”：
   - 点击按钮打开模态窗口。
   - 绘制两条曲线：
     - 无护甲：使用现有伤害公式 `damage(d)`。
     - 有护甲：`damage(d) * 0.5`。
   - 横坐标 label：距离；纵坐标 label：伤害。
   - 悬浮在曲线上显示 `(x, y)`。
   - 点击第一个点时，用 `2px` 半径红点标记该位置（之后不再更新点）。

**非目标：**

- 不实现 `.txt` 的世界坐标到雷达图像素的“像素级精确对齐”（仅用于刻度/坐标的比例与读数需求）。
- 不引入 React/Vue/构建工具；仍保持单文件原生 HTML/CSS/JS。

## 设计决策

1. 伤害曲线与刻度绘制选择 `canvas`
   - 便于随缩放/平移重绘。
   - 支持 tooltip 需要的点/曲线命中近似计算。
2. 主/副刻度生成策略
   - 先根据“目标主刻度在屏幕上的视觉间距”（例如约 110px）反推应选的主刻度世界长度。
   - 在候选集合 `1/2/5 * 10^n` 中选择与目标最接近的一档作为主刻度间距。
   - 根据主刻度的“首位数字”规则确定副刻度间距：`1/5 => major/5`，`2 => major/4`。
   - 每次 `redraw()` 使用当前 `tx/ty/zoom/S` 计算可视区域，遍历在可视范围内落点的刻度并绘制。
3. 伤害曲线坐标与命中逻辑
   - 将距离范围定义为 `[0, 3.5 * baseDamage]`（与页面“远距离伤害为 0”的逻辑保持一致）。
   - 将伤害范围定义为 `[0, baseDamage]`，两条曲线分别用无护甲 `damage(d)` 与有护甲 `damage(d)*0.5` 绘制。
   - 对悬浮点：根据鼠标 x 将距离 d 反算，然后计算两条曲线在该 d 的 y，取离鼠标 y 的最小差值来判断“悬浮在哪条曲线附近”，并显示 tooltip `(d, damage)`。
   - 对点击点：若尚未标记首点，则将命中更近的一条曲线的值写入 `damageCurvePoint` 并重绘；后续点击不再改点。

## 公式来源（与现有页面保持一致）

- 无护甲伤害：
  - 当 `d > 3.5 * baseDamage`：`damage = 0`
  - 否则：`damage = baseDamage * exp(-0.5 * ( (6*d) / (7*baseDamage) )^2 )`
- 有护甲伤害：
  - `damageArmor = damage * 0.5`
