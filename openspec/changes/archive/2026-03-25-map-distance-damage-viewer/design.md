## 上下文

项目需在 `public` 下维护地图资源：**源** Valve 概览 `.txt` 可用于转换（可仍位于 `public/map` 等约定路径）；**归档后** 的元数据放在 **`public/json/`**（每图一个 `.json`），雷达图放在 **`public/png/`**（如 `de_dust2_radar_psd.png`）。页面运行时只从 `public/json` 与 `public/png` 加载，不将 JSON 与 PNG 混放在未区分的目录。本变更新增单页工具，使用原生 HTML/CSS/JS 与 [Pico CSS](https://picocss.com/)，无打包步骤，便于直接由静态服务器打开。

## 目标 / 非目标

**目标：**

- 左右分栏布局：左侧为正方形地图视口（高度约等于视口高度，宽度等于高度）；右侧为 Pico 样式的地图选择与三行只读数值（距离、基本伤害、计算伤害）。**距离行**：在「游戏单位」数值后**以括号**展示米制距离，**1 unit = 25.4 mm**（即米 = 游戏单位 × 0.0254）；页面说明中可简要注明换算关系。
- 地图源：下拉选项对应 7 张地图基名（与源 `.txt` 基名一致，如 `de_dust2`）；选中后从 **`public/png/`** 加载 `*_radar_psd.png`，从 **`public/json/`** 加载同基名 `.json`。**元数据**：实现时先将各图 `.txt` 转为 JSON 并**写入 `public/json/`**；PNG **复制或移动归档至 `public/png/`**；页面 **只从上述两目录** 读取资源。
- **铺满与白边**：正方形视口内地图必须**始终完全覆盖**画面，**永不露出白边**（或页面背景）。实现要点：最小缩放不得低于「整图至少铺满视口」的尺度（与 `object-fit: cover` 在方形容器+方图时的含义一致；方图+方视口时与「完整落入」尺度相同，但非方形容器时需以「覆盖」为准）；平移时必须**夹紧**偏移量，使缩放后任意时刻地图边缘均不「缩进」到视口内形成空隙。滚轮缩放后若会产生露空，应同步修正平移或禁止该缩放步。
- 在视口左下角叠加当前「倍率」文案：`1px = X unit`，其中 \(X = (1024 / S) \times \text{scale}\)，\(S\) 为当前视口边长（px），`scale` 来自当前地图的 **JSON**（若缩放/裁切导致「有效显示边长」与 \(S\) 不一致，实现须在代码中统一 \(S\) 的定义并与倍率一致）。
- 指针悬停于左侧地图区域时，滚轮调整缩放：最小缩放为「整图刚好适配视口」；最大缩放为 `1px = 1` 游戏单位（即倍率文案中 \(X = 1\)）。缩放围绕视口中心或指针—需在实现中固定一种并在代码注释中说明。
- **左键拖动平移**：用户在地图区域**按住左键并拖动**时平移地图（与 Canvas/SVG 叠加层同步）。与测距的区分：**单击**（移动未超阈值即判定为点击）在释放时落点圆心；**按住并移动超阈值**则进入拖动平移，不触发该次「单击落点」。首次单击释放后，在**未按住按键**时移动鼠标更新测距圆半径。
- **参考圆（首次有效单击后）**：圆心均为第一次单击位置。**红圈**：半径（游戏单位）\(d_{100} = \text{baseDamage} \cdot \sqrt{\ln\left(\frac{100}{\text{baseDamage}}\right) \cdot \left(-\frac{2 \cdot 7^2}{6^2}\right)}\)（即 \(\sqrt{\mathrm{LN}(100/\text{baseDamage}) \cdot (-2 \cdot 7 \cdot 7 / 6 / 6)} \cdot \text{baseDamage}\)），要求 \(100 \le \text{baseDamage}\)；若 \(\text{baseDamage} < 100\) 则无法达到 100 伤害，红圈不绘制或显示不可用状态（实现任选其一并在代码注释说明）。**绿圈**：半径（游戏单位）\(d_{\text{green}} = \text{baseDamage} \times 3.5\)。两圆须按当前倍率 \(X\) 换算为屏幕像素半径绘制。
- 测距圆：第一次**单击**落点记录圆心（与参考圆共心）；**释放后**在无按键按下时移动鼠标，绘制以圆心为中心、当前指针为圆周上一点的圆；半径对应的**距离**（游戏单位）= 像素距离 × 当前倍率 \(X\)。
- 第二次**单击**左键（在已有未固定圆的前提下）：固定圆与当前数值，不再随鼠标更新距离与伤害。
- **右键**：清除圆、圆心及固定状态，可重新测距。
- 伤害：`baseDamage` 来自当前地图 `.json` 的 `c4_base_damage`。若游戏单位距离 \(d > 3.5 \times \text{baseDamage}\)，则展示伤害为 **0**（与绿色参考圆半径一致：圆外即无伤害）。否则 \(\text{damage} = \text{baseDamage} \times e^{-\frac{1}{2}\left(\frac{6 \times d}{7 \times \text{baseDamage}}\right)^2}\)。

**非目标：**

- 不实现旋转地图、`pos_x`/`pos_y` 世界坐标与点击点的精确对齐（仅雷达图与比例示意）。
- 不引入 React/Vue、构建工具或后端 API。

## 决策

| 决策 | 说明 | 备选 |
| --- | --- | --- |
| 使用 `<canvas>` 或叠加层绘制圆与命中测试 | 圆需随缩放平移更新，Canvas 或绝对定位 SVG 覆盖在 `<img>` 上，同步 CSS/变换矩阵 | 纯 CSS 无法实现动态半径圆与精确像素测距 |
| `.txt` → `.json` | 实现第一步将各图 txt 转为 json（脚本或一次性转换），运行时 `fetch` JSON | 页面不直接解析 Valve txt；JSON 便于类型稳定与后续扩展字段 |
| 参考圆颜色 | 红 = damage=100 等伤害线；绿 = baseDamage×3.5 半径 | 与测距用动态圆样式区分（线型/颜色） |
| 距离米制 | 括号内 `(\mathrm{m})`，`米 = 游戏单位 × 25.4 / 1000` | 与 spec 中 1 unit = 25.4 mm 一致 |
| 单击 vs 拖动 | 用移动阈值区分：超阈值视为平移，否则 mouseup 视为单击落点 | 避免左键同时承担拖动与落点时的冲突 |
| 铺满与夹紧 | `cover` 式最小缩放 + 平移边界约束 | 满足「永不露白边」 |
| 页面入口 | 单文件 `public/map-range.html`（或项目约定的 `public/` 根路径）并链入 Pico CDN | 多文件拆分可在 tasks 中列为可选 |
| 右键菜单 | `contextmenu` 上 `preventDefault` 以用于清除圆 | 保留浏览器默认菜单会与「右键清除」冲突 |

## 风险 / 权衡

- **[风险]** 各图 PNG 文件名若不完全遵循 `de_xxx_radar_psd.png`，加载会失败 → **缓解**：在 tasks 中核对 7 张文件实际命名并写死映射表或单一命名规则。
- **[风险]** txt→json 转换若与源文件不同步 → **缓解**：转换脚本纳入仓库或在 `tasks` 中约定更新流程；页面以 JSON 为准。
- **[权衡]** 缩放锚点选中心更简单，但与部分用户「绕指针缩放」习惯不一致 → 在设计中二选一并在规范中写明。

## 迁移计划

新增静态文件即可；无数据库与配置迁移。回滚为删除新增 HTML/脚本引用。

## 开放问题

- 缩放锚点采用视口中心还是指针位置：建议在实现阶段统一为**视口中心**以降低与 `object-fit` 联动的复杂度，若与产品预期不符可再改为指针锚点。
