## 新增需求

### 需求:页面布局与地图视口尺寸

系统必须使用原生 HTML 结构，并引入 Pico CSS 作为唯一 UI 框架。页面必须分为左右两栏：左侧区域必须用于地图展示，其高度必须基本占满浏览器可视区域高度，宽度必须等于其高度，从而呈现正方形视口。

#### 场景:视口为正方形且占满高度

- **当** 用户在桌面浏览器中打开页面且视口高度足够
- **那么** 左侧地图展示区域必须表现为正方形，且高度必须接近 100vh（允许为滚动条或边距留出极小余量）

### 需求:地图选择与图像加载

系统必须在右侧栏最上方提供下拉选择框，选项必须对应下列七个地图基名（与源 Valve 概览 `.txt` 基名一致：`de_ancient`、`de_anubis`、`de_dust2`、`de_inferno`、`de_mirage`、`de_nuke`、`de_overpass`）。用户必须能够选择其中一张地图；选择后左侧必须从 **`public/png/`** 目录加载与该地图对应的 PNG 雷达图（与现有资源命名一致，例如 `public/png/de_dust2_radar_psd.png`）。图像原始分辨率必须为 1024×1024 像素。地图图层必须**始终铺满**左侧正方形展示框；**禁止**在展示框内出现因未覆盖而产生的**白边**或底层背景露空（允许通过裁切图像边缘实现铺满；平移与缩放后仍须满足本条）。

#### 场景:切换地图更新图像

- **当** 用户从下拉框选择另一地图名称
- **那么** 左侧必须加载并显示该地图对应的 PNG，且必须重置或按规范更新缩放与测距状态（见清除与固定需求）

### 需求:地图元数据为 JSON

系统必须从 **`public/json/`** 目录下与地图同名的 `.json` 文件加载元数据（例如 `public/json/de_dust2.json`）。该 JSON 必须由实现流程中的 **txt 转 JSON** 步骤从各源 `.txt` 生成或维护，并**归档**到 `public/json/`；**不得**在运行时以未结构化的 txt 作为唯一数据源。JSON 至少必须包含 `scale` 与 `c4_base_damage`（数值类型），且必须能唯一对应所选地图基名。雷达 PNG 必须**归档**在 **`public/png/`** 目录（与 `public/json` 并列，均在 `public` 下），页面不得依赖将 PNG 与 JSON 混放在未区分的单一目录中。

#### 场景:加载 JSON 元数据

- **当** 用户选择某地图且存在对应的 `.json`
- **那么** 系统必须解析 JSON 并获得 `scale` 与 `c4_base_damage` 用于倍率与伤害计算

### 需求:倍率展示与滚轮缩放

系统必须从当前地图的 `.json` 中读取 `scale` 数值。系统必须在左侧展示框左下角显示当前倍率文案，格式必须能够理解为「每屏幕像素对应的游戏单位数」，且对于边长为 \(S\) 像素的正方形展示框与 `scale` 值，展示所依据的倍率 \(X\) 必须满足 \(X = (1024 / S) \times \text{scale}\)（与用户在需求中给出的示例一致）。当用户鼠标悬浮在左侧地图区域上时，系统必须支持滚轮放大或缩小地图显示；缩放后 \(S\) 或等效显示尺寸与倍率 \(X\) 必须同步更新。地图显示缩放必须不得小于「整图刚好适配展示框」时的状态；必须不得大于「\(1\) 屏幕像素对应 \(1\) 游戏单位」即 \(X = 1\) 时的状态。

#### 场景:悬停时滚轮改变缩放

- **当** 鼠标指针位于左侧地图展示区域内且用户滚动鼠标滚轮
- **那么** 地图的显示缩放必须在上述最小与最大界限内变化，且左下角倍率文案必须反映当前 \(X\)，且缩放后视口内**不得**出现白边或未覆盖区域（见「地图铺满与白边」需求）

### 需求:左键拖动平移地图

系统必须支持用户在左侧地图展示区域内**按住鼠标左键并移动**（拖动）时**平移**地图图层（与滚轮缩放、测距圆叠加层同步变换）。平移量必须与用户拖动位移一致或按实现采用等效映射，且平移后必须仍满足「地图铺满与白边」需求中的不露空约束。

#### 场景:按住左键拖动

- **当** 用户在地图展示区域内按下左键并按住不放且移动鼠标
- **那么** 地图必须随拖动平移，且视口内不得露出白边或未覆盖区域

### 需求:地图铺满与白边

**无论**当前缩放比例、平移偏移或窗口尺寸如何，左侧正方形展示框内**必须**被地图像素完全覆盖，**禁止**出现白边、未绘制区域或页面背景从地图与视口之间露出。实现必须通过「最小缩放不低于铺满视口所需尺度」与「平移夹紧」等机制保证；若用户操作会导致露空，系统必须拒绝该操作或自动修正平移/缩放。

#### 场景:缩放与平移后不露白

- **当** 用户完成滚轮缩放或左键拖动平移（或二者组合）
- **那么** 展示框内必须仍无白边或未覆盖区域

### 需求:圆形测距与交互状态

系统必须通过**单击**左键（按下并释放，且移动距离未超过实现约定的**拖动阈值**，因而被判定为点击而非拖动）在地图上记录圆心位置。**在用户第一次有效单击并释放之后**，系统必须立即在以该点为圆心的位置绘制**红色**参考圆与**绿色**参考圆：红色圆的半径（游戏单位）\(d_{100}\) 必须按下式计算：\(d_{100} = \text{baseDamage} \cdot \sqrt{\ln\left(\frac{100}{\text{baseDamage}}\right) \cdot \left(-\frac{2 \cdot 7^2}{6^2}\right)}\)（即 \(\sqrt{\ln(100/\text{baseDamage}) \cdot (-2 \cdot 7 \cdot 7 / 6 / 6)} \cdot \text{baseDamage}\)；自然对数），且仅当 \(\text{baseDamage} \ge 100\) 时该红圈必须存在；若 \(\text{baseDamage} < 100\)，系统必须不绘制红圈或明确标示该参考不可达（任选一种实现方式）。绿色圆的半径（游戏单位）必须为 \(\text{baseDamage} \times 3.5\)。两参考圆须随地图缩放与倍率 \(X\) 正确换算为屏幕像素。

在第一次有效单击**释放之后**、用户**未按住任何鼠标键**移动指针的过程中，系统必须绘制测距圆，其圆心必须为第一次单击位置，半径必须由圆心到当前指针位置的欧氏距离（像素）确定。系统必须计算并展示**距离**（游戏单位）：该距离必须等于像素欧氏距离乘以当前倍率 \(X\)。用户第二次**单击**左键时，若当前存在未固定的测距圆，系统必须将该测距圆与圆心固定，且必须停止随指针更新距离与伤害数值；**参考红圈与绿圈**必须保持显示直至清除。用户按下鼠标右键时，系统必须清除测距圆、圆心、固定状态及**两条参考圆**，并必须允许重新开始测距。

#### 场景:右键清除

- **当** 用户在地图区域点击鼠标右键
- **那么** 测距圆、圆心、红/绿参考圆必须被清除，且距离与伤害展示必须恢复为未测距时的约定显示（例如空或 `--`）

### 需求:距离米制换算显示

系统必须在展示**游戏单位**测距距离时，于**同一行**在「游戏单位」文案之后，以**括号**形式附加展示**米**制距离。换算必须固定为：**1 游戏单位 = 25.4 mm**（即米制长度 = 游戏单位距离 × \(25.4 / 1000\) m）。括号内必须包含以 **m** 为单位的数值（实现可采用合理小数位数，例如保留两位小数）。无有效测距时，不得展示括号及米制内容。

#### 场景:括号内显示米

- **当** 存在有效的当前测距距离（游戏单位）
- **那么** 距离行必须呈现「数值 + 游戏单位文案 + ` (… m)`」形式，其中括号内为按 1 unit = 25.4 mm 换算的米制距离

#### 场景:无测距时不显示米括号

- **当** 用户尚未落点测距或已清除测距
- **那么** 距离行不得包含米制括号内容

### 需求:元数据读取与伤害计算

系统必须从当前地图的 `.json` 中读取 `c4_base_damage` 作为基本伤害。系统必须在地图选择框下方第二行展示当前距离（游戏单位），并满足上条「距离米制换算显示」需求；第三行展示基本伤害，第四行展示计算伤害。设 **距离** 为游戏单位 \(d\)，**基本伤害** 为 \(\text{baseDamage}\)。当 \(d > 3.5 \times \text{baseDamage}\) 时，第四行展示的**计算伤害必须**为 \(0\)。当 \(d \le 3.5 \times \text{baseDamage}\) 时，计算伤害必须按下式计算：\(\text{damage} = \text{baseDamage} \times e^{-\frac{1}{2}\left(\frac{6 \times d}{7 \times \text{baseDamage}}\right)^2}\)。当圆被固定后，距离与伤害数值必须保持固定直至右键清除或更换地图。

#### 场景:显示基本伤害与公式伤害

- **当** 用户已选择地图且 `.json` 中存在有效的 `c4_base_damage`
- **那么** 第三行必须显示该基本伤害值，且第四行在存在有效距离时必须显示按上条规则计算的结果

#### 场景:远距离伤害为零

- **当** 当前测距距离 \(d\) 严格大于 \(3.5 \times \text{baseDamage}\)
- **那么** 第四行计算伤害必须显示为 \(0\)
