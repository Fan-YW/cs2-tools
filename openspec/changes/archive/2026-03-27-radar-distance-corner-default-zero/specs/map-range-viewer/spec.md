## ADDED Requirements

### Requirement: 地图右下角距离叠加展示

系统必须在左侧地图展示框**右下角**以**叠加层**形式展示当前用于测距与公式计算的**游戏单位距离**及米制换算。该叠加层文案的**字体、字号、颜色、内边距、对齐方式及背景/描边（若存在）**必须与同一展示框**左下角**「每屏幕像素对应游戏单位」倍率文案的实现对用户可见效果一致（允许通过复用同一样式类或 design token 达成）。展示格式必须为：`游戏单位距离的十进制表示` + 项目约定的游戏单位文案 + 空格 + `(` + 按 **1 游戏单位 = 25.4 mm** 换算的米制数值（实现 SHALL 采用固定小数位数，例如保留两位小数）+ ` m)`。

#### Scenario: 右下角与左下角视觉一致

- **当** 用户已加载地图且地图展示框同时显示左下角倍率与右下角距离
- **那么** 两处文案的视觉层级与样式必须一致或刻意复用同一套样式，且右下角必须呈现「数值 + 单位 + ` (xx.xx m)`」形式

#### Scenario: 无圆心时右下角显示零距离

- **当** 用户尚未完成第一次有效测距落点，或已右键清除测距且当前无圆心
- **那么** 右下角距离叠加层仍必须可见，且游戏单位距离必须显示为 0，米制括号必须显示为 ` (0.00 m)`（或实现约定的等价零值格式化）

## MODIFIED Requirements

### Requirement: 圆形测距与交互状态

系统必须通过**单击**左键（按下并释放，且移动距离未超过实现约定的**拖动阈值**，因而被判定为点击而非拖动）在地图上记录圆心位置。**在用户第一次有效单击并释放之后**，系统必须立即在以该点为圆心的位置绘制**红色**参考圆与**绿色**参考圆。绿色圆的半径（游戏单位）必须为 \(\text{baseDamage} \times 3.5\)。两参考圆须随地图缩放与倍率 \(X\) 正确换算为屏幕像素。

红圈目标伤害阈值必须由当前滑块值决定：当 \(HP \ge armor\) 时，目标伤害 \(T = HP + armor\)；当 \(HP < armor\) 时，目标伤害 \(T = HP \times 2\)。红色圆的半径（游戏单位）必须按当前伤害公式对目标伤害 \(T\) 反解到距离 \(d\)（\(\text{damage} = \text{baseDamage} \times e^{-\frac{1}{2}\left(\frac{6 d}{7 \cdot \text{baseDamage}}\right)^2}\)）。若目标伤害不可达（例如 \(T > \text{baseDamage}\) 或 \(T \le 0\)），系统必须不绘制红圈或采用明确的降级显示。**因距离大于 \(3.5 \times \text{baseDamage}\) 时伤害为 0，红圈半径（游戏单位）必须不得大于绿圈半径 \(3.5 \times \text{baseDamage}\)**；若反解得到的 \(d\) 大于该值，必须将红圈半径钳制为与绿圈相同。

在第一次有效单击**释放之后**、用户**未按住任何鼠标键**移动指针的过程中，系统必须绘制测距圆，其圆心必须为第一次单击位置，半径必须由圆心到当前指针位置的欧氏距离（像素）确定。系统必须计算**距离**（游戏单位）：该距离必须等于像素欧氏距离乘以当前倍率 \(X\)。**距离的屏幕展示位置必须满足**「地图右下角距离叠加展示」需求。用户第二次**单击**左键时，若当前存在未固定的测距圆，系统必须将该测距圆与圆心固定，且必须停止随指针更新距离与伤害数值；**参考红圈与绿圈**必须保持显示直至清除。用户按下鼠标右键时，系统必须清除测距圆、圆心、固定状态及**两条参考圆**，并必须允许重新开始测距。

#### Scenario: 红圈阈值随 HP 护甲变化

- **当** 用户调整 HP 或护甲滑块
- **那么** 红圈半径必须按新的目标伤害阈值 \(T\) 实时更新

#### Scenario: 右键清除

- **当** 用户在地图区域点击鼠标右键
- **那么** 测距圆、圆心、红/绿参考圆必须被清除，且用于公式与展示的测距距离必须恢复为 **0**（见「距离米制换算显示」与「元数据读取与伤害计算」），右侧信息区伤害相关展示必须与 \(d = 0\) 一致

### Requirement: 距离米制换算显示

地图右下角叠加层（见「地图右下角距离叠加展示」）必须在**同一行**展示游戏单位距离与米制距离。换算必须固定为：**1 游戏单位 = 25.4 mm**（米制长度 = 游戏单位距离 × \(25.4 / 1000\) m）。**无论**当前是否存在测距圆心，**只要**地图已处于可测距状态，该叠加层必须始终展示游戏单位数值与 ` (… m)` 括号内容；当无圆心或已清除时，游戏单位距离必须为 **0**，米制展示必须为对应的零值格式化（例如 `0.00 m`）。

#### Scenario: 括号内显示米

- **当** 地图已展示且右下角距离叠加层可见
- **那么** 该行必须呈现「数值 + 游戏单位文案 + ` (… m)`」形式，其中括号内为按 1 unit = 25.4 mm 换算的米制距离

#### Scenario: 无圆心时仍显示米括号且为零

- **当** 用户尚未落点测距或已清除测距
- **那么** 右下角距离行必须仍包含米制括号内容，且数值对应 **0** 游戏单位

---

### Requirement: 元数据读取与伤害计算

系统必须从当前地图的 `.json` 中读取 `c4_base_damage` 作为基本伤害。**系统不得在右侧信息区单独占用一行展示当前距离**（距离由地图右下角叠加层展示，并须满足「距离米制换算显示」）。在地图选择框下方，**第二行**必须展示基本伤害，**第三行**展示计算伤害；其下为 HP/护甲滑块与爆炸后状态行。设 **距离** 为游戏单位 \(d\)（用于公式与右侧展示；**无测距圆心或已清除后** \(d\) 必须为 **0**），**基本伤害** 为 \(\text{baseDamage}\)。当 \(d > 3.5 \times \text{baseDamage}\) 时，第三行展示的**计算伤害必须**为 \(0\)。当 \(d \le 3.5 \times \text{baseDamage}\) 时，计算伤害必须按下式计算：\(\text{damage} = \text{baseDamage} \times e^{-\frac{1}{2}\left(\frac{6 \times d}{7 \times \text{baseDamage}}\right)^2}\)。

伤害结算到 HP/护甲时必须遵循：若初始护甲 \(> damage \times 0.5\)，则护甲减少 \(\lfloor damage \times 0.5 \rfloor\)，HP 减少 \(damage \times 0.5\)；否则护甲减少到 0，HP 减少 \(damage - \text{初始护甲}\)。当结算后 HP 小于 0 时，结果行必须使用红色字体显示 `HP:0(0)，护甲：0`；否则结果行必须使用黑色字体显示实际结算结果。

当圆被固定后，距离与伤害数值必须保持固定直至右键清除或更换地图。

#### Scenario: 显示基本伤害与公式伤害

- **当** 用户已选择地图且 `.json` 中存在有效的 `c4_base_damage`
- **那么** 第二行必须显示该基本伤害值，且第三行必须按上条规则显示计算伤害（含 \(d = 0\) 时按公式计算的结果）

#### Scenario: 进入页面未选点时按零距离计算

- **当** 用户已选择地图且元数据有效，但尚未在地图上进行第一次有效测距落点
- **那么** 公式所使用的 \(d\) 必须为 **0**，第三行计算伤害与爆炸后状态结算必须与 \(d = 0\) 一致

#### Scenario: 远距离伤害为零

- **当** 当前测距距离 \(d\) 严格大于 \(3.5 \times \text{baseDamage}\)
- **那么** 第三行计算伤害必须显示为 \(0\)

#### Scenario: 护甲充足分支

- **当** 初始护甲 \(> damage \times 0.5\)
- **那么** 系统必须按「护甲减 \(\lfloor damage \times 0.5 \rfloor\)、HP 减 \(damage \times 0.5\)」结算

#### Scenario: 护甲不足分支

- **当** 初始护甲 \(\le damage \times 0.5\)
- **那么** 系统必须把护甲结算为 0，且 HP 扣减值为 \(damage - \text{初始护甲}\)

#### Scenario: HP 为负红色展示

- **当** 结算后 HP \(< 0\)
- **那么** 系统必须以红色显示固定文本 `HP:0(0)，护甲：0`
