## Context

项目为 Vue 3 地图测距与 C4 伤害计算器；地图左侧为正方形视口，左下角已有「每像素对应游戏单位」倍率 HUD；距离与米制原在右侧信息区展示。本变更将距离 HUD 并入地图叠加层，并在无圆心时统一采用 \(d = 0\) 驱动右侧公式与结算。

## Goals / Non-Goals

**Goals:**

- 在地图容器内右下角渲染距离文案，复用与左下角倍率相同的样式类或同一套 token（避免两套视觉语言）。
- 单一数据源表示「当前用于公式的游戏单位距离 \(d\)」：无圆心 / 清除后与进入页一致为 0；有测距后随指针或固定圆更新。
- 右侧栏去掉重复的「当前距离」行（若仍保留占位需与产品一致；本设计以**不再在右侧单独占行展示距离**为准），基本伤害与计算伤害行顺序上移，计算一律使用上述 \(d\)。

**Non-Goals:**

- 不改变 `scale`、`c4_base_damage`、伤害公式、米制换算常数（1 unit = 25.4 mm）。
- 不改变刻度、拖拽、缩放与地图资源加载逻辑，除非为实现叠加层定位所必需的最小 DOM 调整。

## Decisions

1. **距离状态模型**  
   - 在 composable（如 `useRadarMap`）中显式区分「是否有测距圆心」与「用于 UI/公式的 \(d\)」：无圆心时 \(d := 0\)；有圆心未固定时 \(d\) 为指针相对圆心的游戏单位距离；固定后保持；右键清除后 \(d := 0\) 且无圆心。  
   - **理由**：避免用 `null` 与 `0` 混用导致右侧公式分支不一致；与「默认按 0 计算」一致。

2. **HUD 位置与样式**  
   - 右下角使用与左下角相同的定位策略（如 `position: absolute` + 对称 inset），同一组件或共享 `class`/`variant`，仅文案与数值不同。  
   - **理由**：用户明确要求与左下角「1px = xx unit」一致；减少 CSS 分叉。

3. **右侧栏行序**  
   - 地图选择下一行直接为基本伤害，再下一行为计算伤害（原第三、四行顺延为第二、三行）。  
   - **备选**：保留空行占位——已否决，避免多余空白。

4. **米制括号**  
   - 当 \(d = 0\) 时仍展示 ` (0.00 m)`（或实现约定的两位小数），与「无测距不显示米」的旧行为不同；以本变更 spec 为准。  
   - **理由**：HUD 始终可见且与「按 0 计算」语义一致。

## Risks / Trade-offs

- **[Risk]** 用户习惯从右侧读距离 → **Mitigation**：距离仍在视区内（地图角），格式更完整（含米）。  
- **[Risk]** \(d = 0\) 时计算伤害等于「贴脸」满伤，可能被误解为「未选点也有伤害」→ **Mitigation**：文案上可依赖现有「基本伤害 / 计算伤害」标签；若后续需区分「未选点」与「零距离」，可再增加 UI 提示（非本变更范围）。  
- **[Trade-off]** 清除后显示 `0` 与「从未点击」相同，符合用户要求的统一基线。

## Migration Plan

- 纯前端发布，无数据迁移。回滚：恢复右侧距离行 composable 分支与模板即可。

## Open Questions

- 无（需求已明确：右下角、样式一致、默认 0 与右侧按 0 计算）。
