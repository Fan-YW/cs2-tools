## Why

测距距离目前展示在右侧信息栏，与地图上的倍率提示分离，用户需要在左右之间来回看；进入页面尚未选点时，距离与右侧伤害计算缺少统一的「零基线」语义。将距离与左下角倍率同属地图叠加层并置于右下角，可在同一视区内完成读数；在未选点时用 0 作为距离默认值，可使右侧公式区行为明确、可预期。

## What Changes

- 将**当前测距距离**（游戏单位 + 米制括号）从右侧信息栏移至**左侧地图展示框右下角**叠加显示。
- 右下角距离文案的**视觉样式**与左下角「每像素对应游戏单位」倍率文案一致（字体、字号、内边距、背景/描边等以现有左下角实现为准）。
- 距离展示格式为：`xxxx` + 游戏单位文案 + ` (xx.xx m)`（米制保留合理小数位，与现有换算一致）。
- **进入页面至用户尚未在地图上有有效测距圆心前**，距离视为 **0**；右侧「基本伤害 / 计算伤害」及关联结算按 **d = 0** 计算与展示（在有效地图与元数据已加载的前提下）。
- 用户右键清除测距后的默认行为与「未选点」一致：距离为 0，右侧按 0 距离计算（除非另有产品决策保留「清空」与「零」的区分——本变更采用与未选点相同的 0 基线）。

## Capabilities

### New Capabilities

（无 —— 行为归属现有地图测距与信息栏能力扩展。）

### Modified Capabilities

- `map-range-viewer`：更新「距离展示位置与叠加层」「距离与米制显示在无选点/清除时的默认值」「右侧信息区距离与计算所依据的 d 的约定」等相关需求与场景。

## Impact

- 前端：`useRadarMap` 或等价 composable 中的距离状态、右侧栏布局与计算输入；地图容器上用于左下角倍率与新增右下角距离的叠加层组件或样式。
- 规范：`openspec/specs/map-range-viewer/spec.md` 需增量修订（delta），与实现保持一致。
- 无后端或构建链变更；资源路径与地图 JSON 加载逻辑不变。
