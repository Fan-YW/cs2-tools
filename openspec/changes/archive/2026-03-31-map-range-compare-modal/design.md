## Context

- 目标页面：`src/views/MapRangeView.vue`（右侧表格目前仅支持单行展示，无并排对比）
- 数据来源：对比的“所有数据”以表格当前渲染的数据源为准（即用户在表格里看到的每一列/每一行指标）。
- 交互约束（新增）：
  - 表格最左侧新增「对比」列：未选为 `+`，选中为 `-`
  - 最多同时选中 2 个武器；当选中达到 2 个时自动打开模态窗口
  - 选满 2 个期间禁止选择其它武器（只能在这两个里取消至少一个才可继续选择）

## Goals / Non-Goals

**Goals:**

- 在 `MapRangeView` 表格左侧提供“对比选择”能力，最多选中 2 个条目。
- 当选中 2 个时弹出对比模态窗口，以三列布局展示逐项对比结果：
  - 左列：武器 A 的值
  - 中列：属性名（对应原表格列名/指标名）
  - 右列：武器 B 的值
- 对比高亮规则清晰、一致：
  - 数值可比时：更好=绿色背景，更差=红色背景
  - 相等/无法比较（缺失、非数值等）时：中性样式（不高亮）
- “锁定选择范围”：已选满 2 个时，其它条目的 `+` 按钮不可用（disabled），直到取消其中至少一个。

**Non-Goals:**

- 不在本次变更中重构 `MapRangeView` 的整体数据模型或分页/筛选逻辑（仅在现有表格基础上增量实现对比）。
- 不在本次变更中引入新的 UI 框架（沿用项目现有组件/样式体系）。

## Decisions

### 1) 选择状态的单一来源（Single Source of Truth）

- **Decision**：在 `MapRangeView` 内维护 `compareSelectedIds: string[]`（长度 0-2），并据此驱动：
  - 表格每行的 `+/-` 显示
  - 是否允许点击（当已满 2 且该行未被选中时禁用）
  - 模态窗口打开/关闭
- **Rationale**：避免在表格行对象上打补丁字段，减少与现有计算属性/排序的耦合；也便于实现“锁定选择范围”规则。
- **Alternatives**：
  - 用 `Set`：更适合查找，但在 Vue 模板中展示顺序（A/B）仍需要数组；最终仍需数组化。

### 2) 模态窗口触发与关闭策略

- **Decision**：
  - 当 `compareSelectedIds.length === 2` 时自动打开（`compareModalOpen = true`）
  - 允许用户手动关闭模态，但 **不清空选择**（关闭后仍保持“只能在两者中取消”的锁定规则）
- **Rationale**：用户可能想暂时看原表格；关闭不应破坏已选状态，否则与“关闭后不可选择其他武器”的需求冲突。
- **Alternatives**：
  - 关闭即清空选择：不符合需求。

### 3) 对比项来源与“属性”列生成

- **Decision**：对比项以“表格列定义/表头”作为权威来源，生成一份 `compareRows`：
  - `label`: 属性名（与表头一致，或从 i18n key 获取）
  - `aValue` / `bValue`: 从两条武器行数据取值（与表格渲染用同一套取值/格式化逻辑）
  - `compareType`: `number` / `string` / `na`
- **Rationale**：保证“对表格中的所有数据进行对比”不遗漏；且与用户看到的值一致，避免对比与表格展示不一致。
- **Alternatives**：
  - 直接遍历对象键：会包含非展示字段，也无法稳定排序与命名。

### 4) “更好/更差”的判定规则

- **Decision**：为每个可对比字段指定 `direction`：
  - `higher-is-better`（例如伤害、射速、有效距离等）
  - `lower-is-better`（例如散布/不准确度、恢复时间等）
  - `na`（不可比/纯描述字段：名称、类型等）
  
  当两侧都能解析为有限数值时：
  - `higher-is-better`: 大者为绿，小者为红
  - `lower-is-better`: 小者为绿，大者为红
  - 相等：不高亮
  
  当任一侧缺失或不可解析为数值：不高亮（中性）
- **Rationale**：避免“默认一律越大越好”导致误导；用显式方向控制可解释性。
- **Alternatives**：
  - 只对数值字段做简单大小比较：实现快但容易错。

### 5) UI 实现：复用现有样式与最小侵入

- **Decision**：
  - 表格新增首列 `Compare`：使用按钮样式（或最小化 `<button>` + 现有 token）
  - 模态窗口优先使用项目现有 modal/dialog 组件；若无，则在 `MapRangeView` 内实现一个简单的可访问性最低可行模态（ESC 关闭、遮罩、聚焦管理）。
  - 高亮用 CSS class：`better` / `worse` / `neutral`，颜色使用主题变量或接近现有语义色。
- **Rationale**：减少引入依赖与视觉不一致风险。

## Risks / Trade-offs

- **[对比方向定义不全]** → Mitigation：在 tasks 中列出“字段映射表”并逐项校对；对未知字段默认 `na` 不高亮，避免误导。
- **[模态可访问性不足]** → Mitigation：优先复用已有 Dialog 组件；若自实现，至少支持 ESC、点击遮罩关闭、Tab 限制在模态内。
- **[表格字段格式化与对比解析不一致]** → Mitigation：对比使用“原始数值”进行比较，展示仍用原表格格式化；对比行同时保留 `raw` 与 `display`。
