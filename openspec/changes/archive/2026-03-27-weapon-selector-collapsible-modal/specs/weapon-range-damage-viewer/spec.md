## ADDED Requirements

### Requirement: 武器选择区可折叠展示

系统必须将按 `weaponType` 分组的武器 Tag 多选区域（含每行批量按钮与 Tag 列表）封装为独立 UI 单元（组件或等价结构）。该单元必须支持展开与折叠两种状态。折叠状态下，系统必须仅展示固定文案「选择武器」以及表示可向下展开的图标（或等价视觉），不得占用展开态下 Tag 列表的纵向空间。展开状态下，系统必须继续满足本规范中「武器 Tag 分组多选与批量操作」需求的全部场景（分组、批量文案、全选/全清、Tag 点击切换选中）。用户必须能够通过点击标题行（或专用按钮）在折叠与展开之间切换。

#### Scenario: 折叠时仅显示标题与展开 affordance

- **WHEN** 用户将武器选择区置于折叠状态
- **THEN** 界面必须显示「选择武器」文案与向下展开图标（或等价）
- **并且** 不得显示各 `weaponType` 行内的武器 Tag 与批量按钮

#### Scenario: 展开时行为与现有一致

- **WHEN** 用户将武器选择区置于展开状态
- **THEN** 武器必须仍按 `weaponType` 分行展示，且每行批量按钮与 Tag 多选行为必须符合「武器 Tag 分组多选与批量操作」需求中的场景

### Requirement: 表格武器名可打开详情模态

系统必须使伤害结果表格中每一数据行第一列（武器展示名称）成为可激活控件。当用户激活该控件时，系统必须打开模态窗口（对话框）。模态必须从已加载的武器 JSON 记录中读取该武器数据并展示静态属性；不得依赖额外网络请求（数据须来自当前页面已加载的同一份武器数据）。模态必须提供关闭方式（例如关闭按钮、点击遮罩或 Escape，以实现与 Reka UI 组件一致为准）。除第一列外，表格其余伤害数值单元格不得因本需求变为打开同一模态的入口。

#### Scenario: 点击武器名打开模态

- **WHEN** 用户在结果表中点击某一行的武器名称单元格
- **THEN** 系统必须显示模态窗口
- **并且** 模态内容必须对应该行所选武器

#### Scenario: 关闭模态

- **WHEN** 用户以组件支持的关闭方式关闭模态
- **THEN** 模态必须消失且测距与武器选中状态保持不变

### Requirement: 武器详情模态中的 JSON 字段展示

模态内必须按下列规则展示属性（数值来自该武器记录的 `columns` 或等价结构，键名匹配须大小写不敏感；缺失或非有限数时展示「—」或省略该项，但不得崩溃）：

- **护甲衰减**：`WeaporArmorRatio`（或实现中已支持的同义列）除以 2，结果以百分比形式展示。
- **基本伤害**：`Damage`。
- **距离衰减**：`RangeModifier`。
- **爆头倍率**：`HeadshotMultiplier`（或实现中已支持的同义列名）。
- **射速**：\(60 / \text{CycleTime}\)，单位为 rpm（`CycleTime` 须为正有限数）。
- **击杀奖励**：以货币形式展示，前缀为「$」加上 `KillAward` 的数值。
- **最大速度**：`MaxPlayerSpeed`，单位为 unit/s。
- **静步最大速度**：`MaxPlayerSpeed × 0.52`，单位为 unit/s。
- **蹲走最大速度**：`MaxPlayerSpeed × 0.34`，单位为 unit/s。
- **弹夹容量**：来自表内弹容列（例如 `clip_size` 或导出脚本使用的等价列名）。
- **备弹**：展示 `m_nPrimaryReserveAmmoMax` 的数值；若 `m_bReserveAmmoAsClips` 为 true，则该数值的单位文案为「弹夹」，否则为「子弹」。
- **武器价格**：`WeaponPrice`。
- **最远射击距离**：`Range`。
- **类型**：`WeaponType`（列内字符串，如 SubMachinegun）。
- **霰弹数**：仅当 `Bullets` 严格大于 1 时展示该项及其标签；为 1 或缺失时不展示霰弹数行。

#### Scenario: 模态展示与 JSON 一致

- **WHEN** 用户打开某武器的详情模态且该武器在 JSON 中具有上述列的有效取值
- **THEN** 模态必须包含上述各字段的格式化展示，且射速、速度与百分比等必须符合本节公式与单位说明

#### Scenario: 单弹丸武器不展示霰弹数

- **WHEN** 某武器的 `Bullets` 为 1 或缺失
- **THEN** 模态不得展示「霰弹数」条目

#### Scenario: 多弹丸武器展示霰弹数

- **WHEN** 某武器的 `Bullets` 为大于 1 的有限数
- **THEN** 模态必须展示霰弹数及其数值
