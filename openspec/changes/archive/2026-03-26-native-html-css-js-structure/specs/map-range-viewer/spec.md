## MODIFIED Requirements

### 需求:地图选择与图像加载

系统必须在右侧栏最上方提供下拉选择框，选项必须对应下列七个地图基名（与源 Valve 概览 `.txt` 基名一致：`de_ancient`、`de_anubis`、`de_dust2`、`de_inferno`、`de_mirage`、`de_nuke`、`de_overpass`）。用户必须能够选择其中一张地图；选择后左侧必须从 **`public/png/`** 目录（以静态站点根为基准的仓库路径；当工具页面位于 `pages/` 等子目录时，实现必须通过相对 URL 正确解析到该目录，例如自 `pages/map-range.html` 使用 `../public/png/`）加载与该地图对应的 PNG 雷达图（与现有资源命名一致，例如 `de_dust2_radar_psd.png`）。图像原始分辨率必须为 1024×1024 像素。地图图层必须**始终铺满**左侧正方形展示框；**禁止**在展示框内出现因未覆盖而产生的**白边**或底层背景露空（允许通过裁切图像边缘实现铺满；平移与缩放后仍须满足本条）。

#### 场景:切换地图更新图像

- **当** 用户从下拉框选择另一地图名称
- **那么** 左侧必须加载并显示该地图对应的 PNG，且必须重置或按规范更新缩放与测距状态（见清除与固定需求）

### 需求:地图元数据为 JSON

系统必须从 **`public/json/`** 目录（以静态站点根为基准的仓库路径；当工具页面位于 `pages/` 等子目录时，实现必须通过相对 URL 正确解析到该目录，例如自 `pages/map-range.html` 使用 `../public/json/`）下与地图同名的 `.json` 文件加载元数据（例如 `de_dust2.json`）。该 JSON 必须由实现流程中的 **txt 转 JSON** 步骤从各源 `.txt` 生成或维护，并**归档**到 `public/json/`；**不得**在运行时以未结构化的 txt 作为唯一数据源。JSON 至少必须包含 `scale` 与 `c4_base_damage`（数值类型），且必须能唯一对应所选地图基名。雷达 PNG 必须**归档**在 **`public/png/`** 目录（与 `public/json` 并列，均在 `public` 下），页面不得依赖将 PNG 与 JSON 混放在未区分的单一目录中。

#### 场景:加载 JSON 元数据

- **当** 用户选择某地图且存在对应的 `.json`
- **那么** 系统必须解析 JSON 并获得 `scale` 与 `c4_base_damage` 用于倍率与伤害计算
