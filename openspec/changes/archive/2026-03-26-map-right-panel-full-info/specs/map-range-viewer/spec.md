## MODIFIED Requirements

### 需求:页面布局与地图视口尺寸

系统必须使用原生 HTML 结构，并引入 Pico CSS 作为唯一 UI 框架。页面必须分为左右两栏：左侧区域必须用于地图展示，其高度必须基本占满浏览器可视区域高度，宽度必须等于其高度，从而呈现正方形视口。在桌面宽屏布局下，右侧信息栏（除左侧正方形地图以外的区域）必须在**水平方向**占满剩余可用空间；**不得**因对信息栏设置过窄的最大宽度而导致主内容区在信息栏右侧出现大面积仅背景或空白的区域（允许与页面根容器一致的极小内边距）。

#### 场景:视口为正方形且占满高度

- **当** 用户在桌面浏览器中打开页面且视口高度足够
- **那么** 左侧地图展示区域必须表现为正方形，且高度必须接近 100vh（允许为滚动条或边距留出极小余量）

#### 场景:宽屏下右栏占满剩余宽度

- **当** 用户在宽屏桌面浏览器中打开页面且视口宽度足以并排显示左侧正方形地图与右侧信息栏
- **那么** 右侧信息栏必须在水平方向上占满除左侧地图列以外的**全部**剩余可用空间，且不得在信息栏与视口右缘之间因仅限制信息栏最大宽度而留下大块空白
