## Context

项目当前在 `src/style.css` 的 `:root` 中定义深色令牌（`--rk-bg`、`--rk-surface` 等），`body` 与壳层、面板依赖这些变量；部分 Reka UI 外观类（如 `.rk-select-content`）仍使用硬编码深色背景。武器伤害结果表在 `WeaponRangeView.vue` 的 scoped 样式中定义，边框与单元格样式基于半透明白色描边，在亮底下需整体替换为深色描边与浅表面。

## Goals / Non-Goals

**Goals:**

- 以 **CSS 变量**为单一事实来源，将默认主题切换为**日间亮色**（白/浅灰背景、深色主文字、可访问对比度）。
- 统一调整依赖 `--rk-*` 的壳层、按钮、输入与面板；对仍硬编码深色的全局类补充变量或覆盖为亮底等价物。
- 提升 **`.weapon-table`**（及同类表格模式）的层次：表头与数据区区分、边框/分隔清晰、内边距与圆角容器协调。

**Non-Goals:**

- 不引入夜间/日间双主题切换器或 `prefers-color-scheme` 多方案（除非后续单独变更）；本次仅将**默认**设为日间。
- 不改变武器计算公式、表格列结构、Reka 组件行为与路由。
- 不要求更换地图雷达图资源；地图视口内部可保持中性深底以保证雷达图对比度（若与全白冲突，优先保证地图可读性，侧栏与页边为亮底）。

## Decisions

1. **令牌驱动亮色**  
   **选择**：在 `:root` 将 `--rk-bg`、`--rk-surface`、`--rk-text`、`--rk-muted`、`--rk-border` 等改为亮底配色，并设 `color-scheme: light`。  
   **理由**：现有大量规则已引用变量，改动面最小。  
   **备选**：为 `html.light` / `html.dark` 两套变量——超出本次「默认日间」范围。

2. **表格样式落点**  
   **选择**：武器表继续在 `WeaponRangeView.vue` 的 scoped CSS 中增强，但颜色尽量使用全局变量（若 scoped 无法访问则新增 `--rk-table-*` 在 `:root`）。  
   **理由**：表格结构仅此处使用，scoped 保持封装；共享令牌避免与壳层脱节。

3. **硬编码深色组件**  
   **选择**：逐项将 `.rk-select-trigger` 背景、`.rk-select-content`、滑块轨道等改为使用变量或 `rgba` 与亮底协调的固定浅色系。  
   **理由**：否则下拉与表单在亮页上会「挖洞」成深色块，视觉不一致。

4. **高亮伤害色**  
   **选择**：保留红色高亮（≥100），必要时微调饱和度以保证在白色背景上仍满足 WCAG 对比度直觉（仍以「红」为语义）。  
   **理由**：与现有产品规范一致，仅作对比度微调而非改需求。

## Risks / Trade-offs

- **[风险]** 全局变量一改，所有页面同时变亮；若某视图有遗漏的硬编码深色，会出现局部「补丁」。  
  **缓解**：实现后全站点一遍（首页、地图工具、武器工具、弹层）。
- **[风险]** 地图视口若保持深灰，与纯白页边对比强烈。  
  **缓解**：可接受为内容区；若需统一，可仅将 `.map-viewport` 改为浅灰边框包裹的「卡片」内深底。

## Migration Plan

- 纯前端样式变更：合并后直接 `vite build` 验证；无数据迁移。
- **回滚**：恢复 `style.css` 与相关 Vue 样式提交即可。

## Open Questions

- 是否在首版将 `.map-viewport` 背景从 `#1a1a1a` 改为浅灰：取决于雷达图对比度，可在实现时目视决定。
