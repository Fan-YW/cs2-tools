## Context

当前武器页使用单选下拉 + 固定 4x8 表格，只能查看单把武器结果；对比多把武器需要频繁切换。用户希望直接按武器类型点击 Tag 多选，并在同一表格中按行展示多把武器结果。该变更只影响武器页右栏交互与结果呈现，不改变地图测距逻辑和伤害公式输入来源。

## Goals / Non-Goals

**Goals:**
- 将武器选择改为按 `weaponType` 分组的 Tag 多选交互。
- 每个类型行增加批量按钮，支持该类型全选/清空，并按状态显示 `All<TypePlural>` / `Clear<TypePlural>`。
- 结果表支持“每个选中武器一行”，最左新增 `Weapon` 列。
- 有护甲三列展示合并值：`伤害(护甲减少)`，其中伤害保留两位小数，护甲减少为整数。

**Non-Goals:**
- 不修改地图缩放、平移、测距线段逻辑。
- 不更改武器 JSON 结构、导出脚本和已有公式定义。
- 不引入新的第三方依赖或 UI 框架。

## Decisions

1. **选择控件改为“类型分组 + Tag 按钮”**
   - 在 `weapon-range-damage.js` 中按 `weaponType` 对武器分组，渲染每行：`批量按钮 + 多个武器 Tag`。
   - 选中状态用 `Set<string>`（weapon id）维护；点击 Tag 时增删 id 并触发表格重算。
   - 批量按钮文案规则：
     - 当该类型未全选：显示 `All` + 类型复数形式；
     - 当该类型全选：显示 `Clear` + 类型复数形式。

2. **结果表改为多武器行**
   - 表头新增第一列 `Weapon`。
   - 每个选中武器对应一行，取消选中即删除该行。
   - 未选中任何武器时显示空态提示或占位行（实现细节可按现有样式决定）。

3. **有护甲列合并显示**
   - 原“护甲减少”独立行取消。
   - 在三列有护甲单元格显示 `XX.XX(XX)`，其中：
     - `XX.XX`：有护甲伤害（两位小数）；
     - `(XX)`：护甲减少（整数，沿用既有计算与取整规则）。

4. **复用现有计算函数**
   - 将当前单武器计算路径抽成“单武器计算 -> 返回结构化结果”，再对选中集合映射渲染，避免公式分叉。

## Risks / Trade-offs

- [Risk] 选中武器较多时表格过长 → Mitigation: 保持表格容器滚动能力，必要时增加表头 sticky（可后续迭代）。
- [Risk] 类型复数文案不规范（如 rifle/rifles）→ Mitigation: 增加类型到复数字典，未知类型回退 `type + s`。
- [Risk] 多选状态与批量按钮状态不同步 → Mitigation: 所有状态更新统一走 `selectedIds` 的单一数据源并集中重渲染。

## Migration Plan

- 前端静态资源直接发布，无数据迁移。
- 回滚方式：恢复 `pages/weapon-range-damage.html`、`css/weapon-range-damage.css`、`js/weapon-range-damage.js` 到变更前版本。
