## ADDED Requirements

### 需求:武器 Tag 分组多选与批量操作

系统必须将武器选择控件实现为按 `weaponType` 分组的 Tag 区域。每个武器类型必须独立成一行（例如第一行为 `Pistol`）。每行第一列必须提供批量按钮：当该类型尚未全选时，按钮文案必须显示 `All` + 该类型复数形式；当该类型已全选时，按钮文案必须显示 `Clear` + 该类型复数形式。点击批量按钮时，系统必须对该类型执行全选或全清。

#### 场景:按类型分组显示 Tag
- **当** 页面成功加载武器数据
- **那么** 武器选择区必须按 `weaponType` 分行展示，且每把武器为可点击 Tag

#### 场景:批量按钮文案切换
- **当** 某类型下并非全部武器被选中
- **那么** 该类型行首按钮必须显示 `All` + 类型复数形式
- **并且** 当该类型下全部武器被选中时，该按钮必须切换为 `Clear` + 类型复数形式

#### 场景:批量全选与全清
- **当** 用户点击某类型行首 `All...` 按钮
- **那么** 该类型下全部武器 Tag 必须变为选中状态
- **并且** 当用户点击 `Clear...` 按钮时，该类型下全部武器 Tag 必须清除选中状态

## MODIFIED Requirements

### 需求:武器 JSON 数据源（由 xlsx 预生成）

武器原始表为 xlsx 内 **`Raw Values`** 工作表（名称匹配策略以实现为准）。**运行时**页面必须仅通过 HTTP 加载 `public/weapon/` 下由 **`manifest.json`** 的 `weaponsJson` 字段指定的 JSON 文件（缺省时为 `weapons.json`）。该 JSON 必须由仓库内脚本（`scripts/weapon-xlsx-to-json.mjs`）从 `manifest.json` 所指定 xlsx 生成。

每条武器记录**必须**满足：

- **`id`**：字符串，取自 **A 列单元格显示文本**（trim），作为稳定标识与选项值，与现有计算查找一致。
- **`displayName`**：字符串，用于 Tag 标签等 UI 展示；**必须**优先采用 **A 列对应单元格的注释（comment）文本**；若该单元格无注释或注释为空，**必须**回退为 **A 列单元格显示文本**（与 `id` 一致或等价）。
- **`weaponType`**：字符串，表示武器类型（例如 `pistol`），语义须与源表分类一致；具体从 xlsx 的列或表内分段规则导出，以实现与源表对齐为准。
- **全列数据**：对 Raw Values 表**首行表头**所定义的**每一列**（除实现中明确跳过的全空列外），必须在记录中持久化该列对该行的值；结构化形式**必须**为键值对象（或等价结构），键为表头（trim），值为自单元格导出的标量（数字/字符串/布尔/null 等与导出逻辑一致）。公式计算所需的 `Damage`、`HeadshotMultiplier`、`RangeModifier`、`WeaporArmorRatio`（及脚本已支持的列名别名）**必须**能从该结构中解析得到；若某列在表头中缺失或该行无效，则该行不得作为有效武器条目输出。

#### 场景:manifest 与 JSON 就绪
- **当** `manifest.json` 存在且 `weaponsJson` 指向的 JSON 已生成并包含至少一把有效武器
- **那么** 页面必须成功加载该 JSON 且能列出武器

#### 场景:武器展示名使用注释
- **当** 某武器 A 列单元格存在非空注释
- **那么** 武器 Tag（或等价列表）中该条目的可见标签必须显示注释文本，且选项值仍必须对应稳定 `id`

#### 场景:无注释时使用单元格文本
- **当** 某武器 A 列单元格无注释或注释为空
- **那么** `displayName` 与可见标签必须使用 A 列单元格文本

#### 场景:加载或格式失败
- **当** JSON 缺失、无法请求或内容不符合约定结构
- **那么** 系统必须在右栏展示可读错误信息（可提示运行 `npm run weapon:json`），且不得抛出未捕获异常导致白屏

### 需求:有护甲伤害（表格第 3 行第 6–8 列）

系统必须仅在三列展示：**头、胸/手臂、腹**（与表格列对齐）。有护甲伤害必须在对应无护甲伤害基础上乘以 \((\text{WeaponArmor}/2)\)。展示格式必须为 `XX.XX(XX)`：前半部分为有护甲伤害（保留 **2 位小数**），括号内为对应部位的护甲减少整数值（按护甲减少公式计算并取整）。

- **头（有护甲）**：\((\text{Damage} \times \text{HeadShotMultiplier} \times R) \times (\text{WeaponArmor}/2)\)
- **胸/手臂（有护甲）**：\((\text{Damage} \times 1.00 \times R) \times (\text{WeaponArmor}/2)\)
- **腹（有护甲）**：\((\text{Damage} \times 1.25 \times R) \times (\text{WeaponArmor}/2)\)

#### 场景:有护甲三列更新
- **当** 武器或距离变化
- **那么** 有护甲三列必须按上式更新，且每个单元格必须展示为 `XX.XX(XX)`

### 需求:表格结构

系统必须将结果表格实现为“表头 + 动态数据行”结构。表头第一列必须为 **Weapon**，其后列按“无护甲（头、胸/手臂、腹、腿）+ 有护甲（头、胸/手臂、腹）”排列。每个被选中的武器必须对应一条数据行，且该行第一列显示武器展示名称。系统必须在武器选中状态变化时同步增删对应数据行。

#### 场景:选中武器新增行
- **当** 用户选中一个未选中的武器 Tag
- **那么** 结果表必须新增且仅新增该武器对应的一行

#### 场景:取消选中武器删除行
- **当** 用户取消选中一个已选中的武器 Tag
- **那么** 结果表必须删除该武器对应的一行，其他武器行保持不变

## REMOVED Requirements
